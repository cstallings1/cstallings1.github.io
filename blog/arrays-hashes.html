Comparing Arrays and Hashes

What's the difference and which one to use?

December 3, 2015

We're on to another programming language this week - Ruby! Last week we talked about CSS and how we can use it to design websites, this week we'll talk about some Ruby concepts which has more to do with data. Arrays and hashes are data structures used in Ruby that can be very helpful for organizing, retreiving, and updating data. I won't get into too much detail in this post but I want to explain what they are and how you can use them in different situations.

Array

An array is a collection of data ordered by index - it looks like a list: 


        array_of_numbers = [1, 2, 3, 4]

        array_of_strings = ["cat", "dog", "bird"]

        array_of_arrays = [[1, 2, 3], [1, 2, 3], [1, 2, 3]]

As you can see, an array can contain numbers, words, or even more arrays.You can also have an array that contains different types of elements, like numbers and words. I say they are ordered by index because each element in the array is assigned an index starting at 0 (notice we don't start counting at 1) and counting up from there. If I wanted to retrieve the second element in the array_of_strings, I would write: 


        array_of_strings[1]

This grabs "dog" from the array. Notice "cat" is at index 0, and dog is at index 1.

Hash

A hash is similar to an array but it doesn't have an index assigned to it. Instead it has keys assigned to values, and you determine what the keys are. I'll show you an example of a hash using my name as data: 


        my_name = {
        :first_name => "Catie",
        :last_name => "Stallings"
        }

        my_name = {
        first_name: "Catie",
        last_name: "Stallings"
      }

If I wanted to retrieve my first name from this hash I would use the key (first_name) to get the associated value (Catie): 


        my_name[:first_name]

And you'll notice I wrote the hash twice with slightly different syntax, I just wanted to show you that there are different ways of creating a hash. You can see it's similiar to an array but you can add more information to a hash. It helps to think of a hash like a dictionary. Another notable aspect of a hash is that the keys must be unique even though the values don't have to be.

Which to use?

Knowing when to use an array or a hash is something you have to feel out for yourself and take into consideration the code you're writing. There aren't any hard and fast rules I know of. But by seeing how these concepts differ you can start to see how they might be useful in different ways. Hashes are probably best used with associative data. If I'm storing my personal information in either an array or a hash, I'll choose the hash and here's why: 


        personal_info_ary = ["Catie", "Stallings", 25, 60462]

        personal_info_hash = {
        :first_name => "Catie",
        :last_name => "Stallings",
        :age => 25,
        :zip => 60462
      }

The hash give a lot more information about this data, the array is a little confusing. This only scratches the surface of what arrays and hashes can do. If you're interested in seeing some awesome things you can do with them check out the Ruby documentation on arrays and hashes.
