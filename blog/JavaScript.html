Objects in Ruby vs JavaScript

Ruby classes and JavaScript constructor functions

December 23, 2015

Last week we took a look at Ruby classes and how we could use them to model real world objects. We saw that we could create a class (like our cat from the previous post) and create methods and attributes to drive behavior. This week we've moved on to JavaScript so I thought it would be interesting to compare Ruby classes with JavaScript constructor functions.

A constructor function is one of the ways you can create an object in JavaScript, similar to Ruby classes. First, let's create a simple Ruby person class and then we'll see how we can acheive the same outcome using a constructor function. 


            class Person
              def initialize(name, age)
                @name = name
                @age = age
              end
            end

            new_person = Person.new("Catie", 25)

Right now all we've done is created a person class that takes a name and age as arguments, then I created an instance of the class with the name "Catie" and age 25 as shown on the last line. Here's how we'd do that using a constructor function in JavaScript: 


            function Person(name, age) {
              this.name = name;
              this.age = age;
            };

            var newPerson = new Person("Catie", 25);

See the difference? In Ruby, we set the parameters as instance variables denoted by the "@". In JavaScript, we set the parameters as property values denoted by the keyword "this". The syntax is slightly different in the last line, but in both cases we're just creating a new instance and setting it to a variable. Now let's add a method that tells us the name and age of our person. Here's how we do it in Ruby: 


            class Person
              def initialize(name, age)
                @name = name
                @age = age
              end

              def hello
                return "Hi, my name is #{@name} and
                I'm #{@age} years old."
              end
            end

            new_person = Person.new("Catie", 25)
            puts new_person.hello

            OUTPUT:
            Hi, my name is Catie and I'm 25 years old.

And here's how we do it using a constructor function: 


            function Person(name, age) {
              this.name = name;
              this.age = age;
              this.hello = function () {
                return "Hi, my name is " + this.name +
                " and I'm " + this.age + " years old.";
              }
            };

            var newPerson = new Person("Catie", 25);
            console.log(newPerson.hello())

            OUTPUT:
            Hi, my name is Catie and I'm 25 years old.

So we created a property called "hello" and set it's value to a function that returns a string with our person's name and age. A method in JavaScript is this process of creating a property whose value is a function. And notice that just as we have to use the "@" before instance variables in Ruby, we have to use "this" before a property in a constructor function. This was a very simplified example but I hope it gave you an understanding of how both Ruby and JavaScript can be object oriented.
